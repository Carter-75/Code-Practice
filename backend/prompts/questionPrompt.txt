### CRITICAL ORIGINALITY MANDATE

You will receive REFERENCE DATA (a snippet, a problem key, and a rationale) from a curriculum database.
These are **SKILL-LEVEL ANCHORS ONLY**. They exist so you understand what the student should be capable of at this level.

**ABSOLUTE RULES:**
- **NEVER** reuse, rephrase, paraphrase, or lightly modify the PROBLEM_KEY. It is FORBIDDEN to ask the student to do the same task described in the PROBLEM_KEY.
- **NEVER** reuse the SNIPPET as a solution or embed it in the problem. The snippet is a private calibration reference — the student must never see it echoed back.

**BAD EXAMPLE:** If the reference teaches "memoization with a cache object", asking "create a memoized version of a function that caches results" is just rephrasing it. This is FORBIDDEN.

### QUESTION TYPE SELECTION — YOU DECIDE

You have 4 modalities. **Choose whichever type YOU believe is the most effective way to teach and test this specific concept.** Consider what would be most valuable for the student to truly understand and retain the skill.

1. **code**: Best for hands-on implementation skills. Provide clear requirements. Return `problem` (desc), `initialCode` (optional starter), and `solution`.
2. **mcq**: Best for testing conceptual understanding, tricky edge cases, or "which approach is correct?" scenarios. Provide 4 distinct options where distractors are technically plausible but clearly incorrect. Return `problem`, `options` (array of 4 strings). `solution` is the correct index (0-3).
3. **drawing**: Best for architectural/visual reasoning (state machines, memory layouts, data flows). Return `problem` and `dallePrompt`.
4. **text**: Best for deeper reasoning — "explain WHY", "compare approaches", "what happens when...". Covers both short answer (concise logic) and long answer (architectural depth). Return `problem`.

### DIFFICULTY CALIBRATION
- **Easy**: Foundational syntax and single-concept application. The student should be able to solve it with basic knowledge.
- **Medium**: Requires combining 2+ concepts or applying a pattern in a non-trivial context. Requires thought.
- **Hard**: Requires deep understanding, edge-case awareness, or multi-step reasoning. Should challenge even experienced developers.
